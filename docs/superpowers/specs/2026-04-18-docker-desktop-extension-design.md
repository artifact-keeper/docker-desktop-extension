# Artifact Keeper Docker Desktop Extension

## Overview

A Docker Desktop extension that provides a one-click install of Artifact Keeper, an enterprise artifact registry supporting 45+ package formats. Users get a fully functional registry running locally with search, scanning, and a web UI embedded directly in Docker Desktop.

**Extension image:** `artifactkeeper/docker-desktop` on Docker Hub
**Target:** Developers evaluating Artifact Keeper, and teams running it locally for development.

## Architecture

### Services

The extension runs a multi-container stack via `docker-compose.yaml` embedded in the extension image.

**Core (always running):**

| Service | Image | Resources | Port |
|---------|-------|-----------|------|
| backend | ghcr.io/artifact-keeper/artifact-keeper-backend | 256MB mem | configurable (default 8080) |
| postgres | postgres:16-alpine | 512MB mem | internal (optional 30432 toggle) |
| web | ghcr.io/artifact-keeper/artifact-keeper-web | 128MB mem | internal (3000) |
| meilisearch | getmeili/meilisearch:v1.12 | 512MB mem | internal (7700) |

**Optional (off by default, toggled via settings):**

| Service | Profile | Image | Resources | Notes |
|---------|---------|-------|-----------|-------|
| trivy | scanning | ghcr.io/aquasecurity/trivy:0.69.3 | 512MB, 1 CPU | vulnerability scanning |
| openscap | scanning | ghcr.io/artifact-keeper/artifact-keeper-openscap | 512MB, 1 CPU | compliance scanning |
| dependency-track | sbom | dependencytrack/apiserver:4.11.4 | 4GB mem, 2 CPU | SBOM analysis, warning badge in UI |
| jaeger | tracing | jaegertracing/all-in-one:1.62 | 256MB | distributed tracing |

Optional service images are pulled lazily on first toggle-on, not bundled in the extension. This keeps the install size small.

### Networking

- Single Docker bridge network: `artifact-keeper-ext`
- Backend binds to `127.0.0.1` by default (local only)
- LAN access toggle changes binding to `0.0.0.0`
- No reverse proxy (Caddy). Backend exposed directly on configurable host port
- Web UI accessed internally by the extension iframe

### Data Persistence

Named Docker volumes for all persistent data:

| Volume | Purpose |
|--------|---------|
| `ak-ext-postgres` | PostgreSQL data |
| `ak-ext-artifacts` | Artifact file storage |
| `ak-ext-meilisearch` | Search index |
| `ak-ext-backups` | Backup files |
| `ak-ext-plugins` | WASM plugins |
| `ak-ext-scan-workspace` | Scanner working directory |
| `ak-ext-dtrack` | Dependency-Track data (when enabled) |

Data survives container restarts and extension updates. Wiped on `docker extension rm`. The settings panel includes a "Backup Data" button that exports volume data to a user-chosen directory.

### Compose Profiles

Single `docker-compose.yaml` using profiles to control optional services:

- Default (no profile): backend, postgres, web, meilisearch
- `scanning` profile: adds trivy + openscap
- `sbom` profile: adds dependency-track + bootstrap init container
- `tracing` profile: adds jaeger

The extension activates profiles based on config. Toggling a service calls the Docker CLI to bring up/down the relevant profile without touching core services.

## Extension UI

### Layout

**Top bar (React, always visible):**
- Artifact Keeper logo + title (left)
- Health status dots: green/red circles for each running service (center)
- Settings gear icon (right) opens slide-out panel

**Main area:**
- Full-width iframe rendering the Next.js web UI from the internal web service
- Loading state with spinner and "Starting Artifact Keeper..." while services boot
- Error state if services fail to start, with container logs accessible

### Settings Panel (slide-out from right)

**Network section:**
- Port: number input, default 8080
- Network access: toggle between "Local only" (127.0.0.1) and "LAN accessible" (0.0.0.0)
- Expose Postgres: toggle to map port 30432 to host for CLI access

**Services section:**
Each optional service gets a row:
- Name and brief description
- RAM estimate badge
- On/off toggle switch
- Dependency-Track row includes a warning: "Requires 4GB+ RAM for vulnerability database"

**Data section:**
- "Backup Data" button: exports all volumes to a zip file at a user-chosen location
- "Reset All Data" button: confirmation dialog, then removes all named volumes and restarts

**Info section:**
- Extension version
- Backend version (from `/health` endpoint)
- Admin credentials: shown on first run with copy button

## Configuration

### Auto-generated Environment (no user input needed)

```
DATABASE_URL=postgresql://registry:registry@postgres:5432/artifact_registry
STORAGE_BACKEND=filesystem
STORAGE_PATH=/var/lib/artifact-keeper/artifacts
BACKUP_PATH=/var/lib/artifact-keeper/backups
MEILISEARCH_URL=http://meilisearch:7700
MEILISEARCH_API_KEY=<auto-generated>
JWT_SECRET=<auto-generated>
BIND_ADDRESS=127.0.0.1:8080
ENVIRONMENT=development
RUST_LOG=info
SKIP_ADMIN_PROVISIONING=false
CORS_ORIGINS=http://localhost:3000,docker-desktop://
```

Scanner URLs injected dynamically when toggled on:
```
TRIVY_URL=http://trivy:8090
OPENSCAP_URL=http://openscap:8091
DEPENDENCY_TRACK_URL=http://dependency-track-apiserver:8080
OTEL_EXPORTER_OTLP_ENDPOINT=http://jaeger:4317
```

### Secrets Handling

- JWT_SECRET and MEILISEARCH_API_KEY generated with cryptographic randomness on first install
- Stored in a Docker config or a secrets volume (not a plaintext file on the host)
- File permissions set to 0600 if file-based
- Admin password auto-generated on first run, displayed once in the extension UI
- Never logged or written to container stdout

### User Settings (persisted in Docker volume)

```json
{
  "port": 8080,
  "bindAddress": "127.0.0.1",
  "exposePostgres": false,
  "services": {
    "meilisearch": true,
    "trivy": false,
    "openscap": false,
    "dependencyTrack": false,
    "jaeger": false
  }
}
```

Stored in a small `ak-ext-config` named volume, read by the extension on startup.

## Container Health

### Docker HEALTHCHECK on every container

| Service | Health Check | Interval | Retries |
|---------|-------------|----------|---------|
| backend | `curl -sf http://localhost:8080/health` | 5s | 10 |
| postgres | `pg_isready -U registry` | 5s | 10 |
| web | `curl -sf http://localhost:3000` | 5s | 5 |
| meilisearch | `curl -sf http://localhost:7700/health` | 5s | 5 |
| trivy | `curl -sf http://localhost:8090/healthz` | 10s | 5 |
| openscap | `curl -sf http://localhost:8091/health` | 10s | 5 |
| dependency-track | `curl -sf http://localhost:8080/api/version` | 15s | 20 |
| jaeger | `curl -sf http://localhost:16686` | 5s | 5 |

### Startup Dependencies

```
postgres (healthy) -> backend (healthy) -> web
                   -> meilisearch
```

Use `depends_on` with `condition: service_healthy`. Postgres must be fully ready before backend starts migrations.

### Shutdown

- `stop_grace_period: 5s` on postgres for clean shutdown
- Docker Desktop handles stopping all containers on quit
- Named volumes persist across restarts

### Health Monitoring in UI

The extension frontend polls container health via `ddClient.docker.cli.exec("inspect", ...)` to check Docker health status. Does not poll HTTP endpoints from the browser (avoids port mapping dependency).

Update interval: every 10 seconds.

## Iframe Embedding

The extension embeds the full Next.js web UI in an iframe.

**Required headers on the web service:**
- Remove `X-Frame-Options` header (or set to `ALLOWALL`)
- Set `Content-Security-Policy: frame-ancestors *`

**Auth approach:**
- Skip cookie-based auth for the embedded context
- Generate a local API key on startup, pass it to the iframe via query parameter
- The backend trusts this key for all operations (local-only, single-user context)

**Navigation:**
- Links that would open new tabs intercepted by the extension wrapper
- External URLs opened in the system browser via `ddClient.host.openExternal(url)`

## Build and Publishing

### Extension Image

Multi-stage Dockerfile:
1. Go builder: compile the extension backend (settings API, health proxy)
2. Node builder: build the React extension UI
3. Final: Alpine with compiled backend, built UI, compose file, metadata, icon

### metadata.json

```json
{
  "icon": "icon.svg",
  "ui": {
    "dashboard-tab": {
      "title": "Artifact Keeper",
      "src": "index.html",
      "root": "/ui"
    }
  },
  "vm": {
    "composefile": "docker-compose.yaml"
  },
  "host": {}
}
```

### Local Testing

```bash
make build-extension          # Build the extension image
make install-extension        # Install into Docker Desktop
docker extension dev ui-source artifactkeeper/docker-desktop:latest http://localhost:3000
docker extension dev debug artifactkeeper/docker-desktop:latest
```

### Publishing

```bash
make push-extension           # Push to Docker Hub (multi-arch: amd64, arm64)
```

Submit to Docker Extensions Marketplace via Docker's partner portal. Users can also install directly: `docker extension install artifactkeeper/docker-desktop`.

## Upgrade Path

- Backend runs SQLx migrations automatically on startup
- Extension stores a version marker in the config volume
- On major version bumps, the extension UI shows a migration notice before starting
- The "Backup Data" button is prominently shown during upgrades
